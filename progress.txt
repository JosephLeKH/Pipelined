## US-002: FastAPI app skeleton with health endpoint and CORS

Created the core backend module files:

- `backend/main.py`: `create_app()` factory with `lifespan` context manager (Motor connect/disconnect), `CORSMiddleware`, and `GET /health` returning `{"status": "ok"}`
- `backend/config.py`: Pydantic `BaseSettings` with all env vars (DB, auth, OpenAI, GitHub, rate limits, CORS origins)
- `backend/database.py`: Motor async client with `connect()`, `disconnect()`, `get_collection()` — one client per process
- `backend/requirements.txt`: all required packages pinned to exact versions
- `backend/pyproject.toml`: pytest config (asyncio auto mode, importlib import mode)
- `backend/tests/conftest.py`: async test client fixture + per-test DB wipe
- `backend/tests/test_health.py`: 2 tests — 200 response body and CORS header

Learnings:
- `backend/calendar/` shadows stdlib's `calendar` module (used by httpx → http.cookiejar). Fixed by adding `backend/conftest.py` that pre-loads the stdlib calendar via `importlib.util.spec_from_file_location` before any third-party import can trigger the collision.
- pytest-asyncio requires `asyncio_default_fixture_loop_scope` set explicitly to avoid deprecation warnings.
- `--import-mode=importlib` in pytest config prevents pytest from inserting directories into sys.path in the legacy way.
