## US-178 — Wrap App.jsx routes with ErrorBoundary for per-route error isolation
**Status:** COMPLETE
**Commit:** feat: US-178 - Wrap App.jsx routes with ErrorBoundary for per-route error isolation

**What was done:**
Added `Link` to the react-router-dom import and imported the existing `ErrorBoundary` component in `frontend/src/App.jsx`. Added a `RouteErrorFallback` component that renders a user-friendly alert with a "Go to Dashboard" `<Link to="/dashboard">`. Wrapped the `<Routes>` block inside `<ErrorBoundary fallback={<RouteErrorFallback />}>` so per-route errors are caught at the routes level rather than only at the root `main.jsx` boundary. All 5 existing `ErrorBoundary.test.jsx` tests pass; `npm run build` succeeds.

**Learnings:**
- The existing `ErrorBoundary` already supports a `fallback` prop — no changes to the component itself were needed; just pass a custom fallback at the call site.
- Using `<Link>` inside the fallback works because `App.jsx` is rendered inside `BrowserRouter` in `main.jsx`, so router context is available even during the error boundary fallback render.
- Pre-existing a11y test failures (axe concurrency errors) are unrelated to this change.
