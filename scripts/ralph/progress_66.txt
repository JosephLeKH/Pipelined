## US-662: Gate LandingPage.jsx FeaturesSection stagger animation with prefers-reduced-motion
**Status:** PASS
**Summary:** Added a `prefersReducedMotion` check inside the IntersectionObserver callback in FeaturesSection. When `window.matchMedia('(prefers-reduced-motion: reduce)').matches` is true, `in-view` is added immediately without any setTimeout delay; otherwise the existing STAGGER_MS setTimeout logic runs unchanged. STAGGER_MS constant and all other useEffect logic left untouched.
**Learnings:** The prefersReducedMotion check should be evaluated inside the observer callback (not once at effect setup time) so it reflects the user's live media query state at the moment the intersection fires.

Story: US-664 - Add FAQ accordion section to Pricing.jsx below the pricing grid
Date: 2026-05-03
Status: COMPLETE

Summary:
Added a native HTML <details>/<summary> FAQ accordion section to Pricing.jsx below the pricing grid (md:grid-cols-2 div). The section uses Tailwind's open: variant (open:pb-4) to add bottom padding only when an accordion item is expanded. Three questions added: "When does Pro launch?", "Can I switch plans?", and "Is there a student discount?". Build passed in 2.75s.

Learnings:
- Tailwind's open: variant works directly on <details> elements to style the expanded state (e.g., open:pb-4).
- Native <details>/<summary> requires no JS and is keyboard-accessible by default (Enter/Space to toggle).
- Used &apos; HTML entity for the apostrophe in "We're" to avoid JSX string escaping issues inside JSX.
