## US-887 - Rate limit the interview prep SSE endpoint
**Status**: ✅ COMPLETED

**Summary**:
Added rate limiting to the interview prep SSE endpoint to prevent API quota exhaustion. Each request can take 30+ seconds and spawn multi-step Gemini agent loops, so limiting to 3/hour per user is essential.

**Changes**:
- Added `Request` to FastAPI imports
- Imported `limiter` from `middleware.rate_limit`
- Added `@limiter.limit("3/hour")` decorator after `@router.get` decorator
- Added `request: Request` as first parameter with `# noqa: ARG001` comment
- Followed exact pattern from `email_integration/router.py` trigger_sync endpoint
