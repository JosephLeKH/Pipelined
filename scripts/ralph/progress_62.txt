## US-632: Add aria-hidden='true' to Plus and Trash2 icons in CalendarEventsList.jsx

**Summary:**
Added aria-hidden='true' to two decorative icons in CalendarEventsList.jsx:
1. Plus icon (line 31) inside Button with both aria-label='Add event' and visible 'Add Event' text — doubly covered, icon is purely decorative.
2. Trash2 icon (line 62) inside icon-only Button with aria-label='Delete event: {event_type} on {date}' — the aria-label is the sole accessible name, icon is decorative.

Both className values were left unchanged. Build verified clean (✓ built in 2.72s).

**Learnings:**
- Icon-only buttons (size='icon', no visible text) are accessible via aria-label alone; the icon inside should always get aria-hidden='true'.

- Two-edit pattern is clean: one Edit call per icon, minimal diff, no risk of cascading changes.
