## US-216 — Replace color palette with Anthropic brand tones
**Date:** 2026-04-23
**Status:** COMPLETE

### Summary
Overhauled frontend/tailwind.config.js to replace the indigo/violet brand palette and cool slate grays with Anthropic's official brand colors.

### Changes
- `brand.*`: replaced indigo hex values (#6366F1 etc.) with Anthropic clay orange shades centered on brand-500 #d97757 and brand-600 #c4673a
- `gray.*`: replaced `colors.slate` alias with explicit warm-gray hex values (gray-50 #faf9f5 through gray-900 #141413)
- Added `accent-blue: #6a9bcc` and `accent-green: #788c5d` as secondary accent colors
- Added semantic surface tokens: `surface-primary`, `surface-secondary`, `surface-tertiary`
- Added semantic border tokens: `border-default` and `border-strong` (warm rgba values)
- Added `dark.*` token group for dark mode: bg #141413, surface #1c1c1a, accent #d97757, text #e8e4de, text-secondary #b0aea5, border rgba(255,255,255,0.08)
- Removed `glow` shadow (indigo-tinted, now gone)
- Updated `card`, `card-hover`, `modal` shadows to use warm rgba(120,100,75,...) tones
- Updated `button` border-radius from 0.5rem to 0.375rem (6px per spec)
- Removed `accent: colors.amber[500]` (was a named single value, not needed)
- `npm run build` passes clean

### Learnings
- Tailwind's `colors.slate` was being aliased to `gray`, which pulled in cool blue-gray tones — replacing with explicit hex values gives precise control
- Dark mode tokens grouped under a `dark` key in `colors` are accessible via `text-dark-text`, `bg-dark-bg` etc., but won't auto-apply — still need class-based dark mode selectors in components
- The `border-default` / `border-strong` rgba values work fine as Tailwind color values and can be used with `border-border-default` class syntax

## US-217 — Update typography to Poppins headings and Lora body
**Date:** 2026-04-23
**Status:** COMPLETE

### Summary
Established Anthropic's Poppins + Lora editorial font pairing as the project's typography foundation. Lora (serif) is now the body font, Poppins (sans-serif) is the heading and UI font.

### Changes
- `frontend/tailwind.config.js`: changed `fontFamily.sans` from `["Inter", ...defaultTheme.fontFamily.sans]` to `["Lora", "Georgia", "serif"]`; added `fontFamily.display: ["Poppins", "Arial", "sans-serif"]`
- `frontend/index.html`: replaced Google Fonts link from Inter to Poppins (400,500,600,700) + Lora (ital and non-ital 400,500,600,700)
- `frontend/src/index.css`: added `@layer base` block setting body to Lora 15px / 1.65 line-height / -0.01em tracking / #141413 color / antialiased; h1-h6 use Poppins with h1 (600, 1.5rem, -0.02em), h2 (600, 1.25rem, -0.025em), h3 (500, 1.125rem); added `@layer utilities` `.font-display` class
- No text-slate-* or text-indigo-* were present in index.css base layer (already clean)
- `npm run build` passes clean

### Learnings
- `@import` for Google Fonts must come before `@tailwind` directives in CSS, so font imports go into `index.html` as `<link>` tags rather than `@import` statements in index.css
- The tailwind.config.js `fontFamily.display` key automatically generates a `font-display` utility class — the explicit `.font-display` in `@layer utilities` provides an escape hatch for non-Tailwind CSS contexts
- Previous iteration auto-committed tailwind.config.js and index.html changes; only index.css needed to be written this session
